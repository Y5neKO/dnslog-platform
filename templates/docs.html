Authentication: All protected endpoints accept access code via query parameter ?code=xxx or header X-Access-Code: xxx.
POST /api/auth Verify access code

Request Body (JSON)

{
  "code": "your_access_code"
}

Response 200

{ "ok": true }

Response 403

{ "error": "wrong code" }
GET /api/random Generate random token domain

Parameters

ParamInRequiredDescription
codequery / headerYesAccess code

Response 200

{ "domain": "abc12345.{{ domain }}" }

Example

curl "http://HOST:{{ web_port }}/api/random?code=your_access_code"
GET /api/records Get records by token

Parameters

ParamInRequiredDescription
codequery / headerYesAccess code
tokenqueryYesToken string (first part of domain)
limitqueryNoMax records (default: 200)

Response 200

[
  {
    "id": 1,
    "domain": "test.abc12345.{{ domain }}",
    "qtype": "A",
    "src_ip": "1.2.3.4",
    "created_at": "2026-06-10 05:00:00"
  }
]

Example

curl "http://HOST:{{ web_port }}/api/records?token=abc12345&code=your_access_code"

Parameters

ParamInRequiredDescription

WS ws://HOST:{{ ws_port }} WebSocket real-time updates

Connection

// 1. Connect
const ws = new WebSocket("ws://HOST:{{ ws_port }}");

// 2. Send auth within 5 seconds
ws.send(JSON.stringify({ "code": "your_access_code" }));

// 3. Receive updates
ws.onmessage = () => {
  // Fetch latest records via GET /api/records
};

Notes

  • Must send auth message within 5s of connection, or socket is closed
  • Server sends "update" string on new record; client should fetch /api/records
  • Auto-reconnect recommended on close
